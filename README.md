# 🚀 Automation Testing Learning Repository

This repository is a **structured, hands-on learning roadmap for
Automation Testing**, designed for QA Engineers who want to transition
from manual testing to automation.

It combines: - 📚 A **guided learning plan** - 🧪 **Practical exercises
and projects** - 📊 **Real test outputs (reports, screenshots, logs)** -
⚙️ CI/CD integration using GitHub Actions

The goal is not just to learn---but to **build a portfolio that proves
real automation skills**.

Included topics in the training plan:
- JavaScript
- TypeScript
- Cypress
- Playwright

# 🍴 How to Use This Repository (Fork It First)

This repository is meant to be **forked and used as your personal
training workspace**.

## 🔹 Step-by-step: How to fork

1. Click the **Fork** button (top-right of this repository)
2. Select your GitHub account
3. Wait for GitHub to create your copy
4. Rename your forked repository:
    - Go to your forked repo
    - Click Settings
    - Under Repository name, change it to:
    ```
    automation-training-[YOUR-NAME]
    ```
    - Example:
    ```
    automation-training-cedric
    ```
5. Clone your fork locally:

``` bash
git clone https://github.com/[YOUR-USERNAME]/automation-training-[YOUR-NAME].git
```
6. Navigate into the project:
``` bash
cd automation-training-[YOUR-NAME]
```
7. (Optional but recommended) Connect to the original template as upstream:
``` bash
git remote add upstream https://github.com/cedric-magno/automation-training-template.git
```
This allows you to pull future updates from the template if needed using 'git fetch upstream' or 'git pull upstream main'. 
8. Navigate  to 'Automation Training Plan.md' then Crtl+H and replace all 'cedric-magno/automation-training' to '[YOUR-USERNAME]/automation-training-[YOUR-NAME]'
9. Start following the learning plan and commit your progress

------------------------------------------------------------------------

## 💡 Why fork instead of cloning directly?

-   You get your **own version** of the repo
-   You can **track your progress publicly**
-   You build a **portfolio recruiters can see**
-   You can customize the structure without affecting the original
